하나의 인자를 받는 프로그램을 만들어라. 
만약 0개 혹은 1개보다 많은 수의 인자가 들어온다면, 프로그램은 "Error: argument"와 개행 문자를 출력해야 한다.

$> ./micro_paint | cat -e
Error: argument$
$> ./micro_paint do not fear mathematics | cat -e
Error: argument$
$> 

만약 파일을 열거나 읽을 때 문제가 발생한다면, "Error: Operation file corrupted"와 개행 문자를 출력해라.

'작업 파일'은 한 줄에 하나의 작업을 포함한다.
한 줄이 틀리면 에러를 일으킨다.
에러가 발생하면 프로그램은 1을 반환해야 한다.
에러가 발생하지 않으면 프로그램은 0을 반환해야 한다.
마지막 줄은 개행을 포함해도 되고 하지 않아도 된다.
줄은 순서대로 읽혀야 하고, 실행 또한 같은 순서로 실행되어야 한다.
줄의 변수 사이에는 최소 하나의 공백이 존재해야 한다.

WIDTH HEIGHT BACKGROUND_CHAR
이것은 무조건 파일의 첫 줄에 존재해야 하며, 우리가 그릴 공간의 크기를 정의한다.
프로그램은 이 그리는 공간 바깥으로 아무것도 표시하지 않아야 한다. 이 줄은 한 번만 나와야 한다.
- WIDTH : 1과 300 사이의 값을 가진다. 그리는 공간의 가로 부분을 말한다.
- HEIGHT : 1과 300 사이의 값을 가진다. 그리는 공간의 세로 부분을 말한다.
- BACKGROUND_CHAR : 모든 빈 공간은 이것으로 채워질 것이다.

r X Y WIDTH HEIGHT CHAR
이 작업은 빈 직사각형을 그릴 것이며, 직사각형의 테두리만 그려질 것이다.
- r: 문자 r (키워드)
- x: 실수 값을 받는다. 직사각형의 왼쪽 위 꼭짓점 좌표를 말한다.
- y: 실수 값을 받는다. 직사각형의 왼쪽 위 꼭짓점 좌표를 말한다.
- WIDTH: 양의 실수값을 받으며, 0이 아니다. 직사각형의 너비값이다.
- HEIGHT: 양의 실수값을 받으며, 0이 아니다. 직사각형의 높이값이다.
- CHAR: 직사각형을 그리는 데 사용될 문자.

R X Y WIDTH HEIGHT CHAR
이 작업은 꽉 찬 직사각형을 그릴 것이다.
- R: 문자 R (키워드)
- x: 실수 값을 받는다. 직사각형의 왼쪽 위 꼭짓점 좌표를 말한다.
- y: 실수 값을 받는다. 직사각형의 왼쪽 위 꼭짓점 좌표를 말한다.
- WIDTH: 양의 실수값을 받으며, 0이 아니다. 직사각형의 너비값이다.
- HEIGHT: 양의 실수값을 받으며, 0이 아니다. 직사각형의 높이값이다.
- CHAR: 직사각형을 그리는 데 사용될 문자.

그리는 공간은 각각 하나의 문자를 포함하는 직사각형으로 나눠질 것이며, 우리는 그것을 픽셀이라고 부를 것이다.
우리는 쉽게 하기 위해, 왼쪽 위 꼭짓점 픽셀만을 사용하여 목표 직사각형이 그리는 공간 안에 있는지를 확인할 것이다.

당신은 our_micro_paint를 사용하여 당신의 작업 파일을 테스트 해볼 수 있습니다.

힌트 :
만약 좌표가 (Xa, Ya)로 정의되고,
직사각형의 왼쪽 위 꼭짓점이 (Xtl, Ytl)이고 오른쪽 아래 꼭짓점이 (Xbr, Ybr)일 때,
만약 Xtl <= Xa <= Xbr이고 Ytl <= Ya <= Ybr이면 그릴 공간 안에 존재하는 것이다.
